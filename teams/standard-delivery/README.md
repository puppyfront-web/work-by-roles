# Standard Delivery Team Configuration

标准的软件交付团队配置，适用于大多数软件开发项目。

## 特点

- **四阶段工作流**：需求分析 → 架构设计 → 实现开发 → 质量验证
- **角色分离**：产品分析师、系统架构师、核心框架工程师、QA 评审员
- **技能导向**：每个角色都有明确的技能要求
- **质量门禁**：每个阶段都有质量检查点
- **文档可选**：默认不生成文档，专注于代码实现（用户无特殊要求不生成）

## 适用场景

- 标准软件项目开发
- 需要严格流程控制的项目
- 团队协作开发
- 快速迭代开发（文档可选）

## 工作流阶段

1. **Requirements & Scope** (产品分析师)
   - 需求收集和分析
   - 范围定义
   - 成功标准制定

2. **Architecture & DSL Design** (系统架构师)
   - 系统架构设计
   - 数据模型定义
   - 接口规范

3. **Minimal Runnable Implementation** (核心框架工程师)
   - 核心功能实现
   - 代码编写
   - 单元测试

4. **Validation & Quality Gates** (QA 评审员)
   - 质量检查
   - 测试验证
   - 文档审查

## 使用方法

### 作为项目默认配置（推荐）

此配置是项目的**标准交付团队配置**，会在项目初始化时自动使用：

```bash
# 初始化项目（自动使用此配置）
workflow init
```

### 创建新的团队实例

```bash
# 基于此配置创建新团队
workflow team create my-project --template standard-delivery --name "我的项目团队"
```

## 自定义建议

根据项目需求，可以：
- 调整阶段顺序
- 添加或删除角色
- 修改技能要求
- 调整质量门禁标准
